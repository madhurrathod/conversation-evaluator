# Conversation Evaluation Benchmark

A production-ready system that scores every turn of a conversation across **276 behavioural and psychological facets** using open-weights LLMs served via Groq.

---

## Overview

| | |
|---|---|
| **Facets scored** | 276 inferrable (cleaned from 398 raw) |
| **Score scale** | 1 – 5 (five ordered integers) |
| **Scoring model** | `llama-3.1-8b-instant` via Groq (open-weights, ≤ 16B) |
| **Generation model** | `llama-3.3-70b-versatile` via Groq (open-weights) |
| **Conversations** | 50 synthetic, 378 turns total |
| **Hard constraints met** | No one-shot prompting · Open-weights ≤ 16B · Scales to 5000+ facets |

```
          ▼
┌─────────────────────┐

│   src/clusterer     │  chunk into even batches of ~15 facets

┌─────────────────────┐

│  src/conversation_  │  generated by llama-3.3-70b-versatile
│  generator          │
└─────────┬───────────┘

          ▼

│  Scoring Pipeline   │  Per turn × per cluster: chain-of-thought prompt
│  src/scorer         │  → llama-3.1-8b-instant → score + reasoning +

          ▼
┌─────────────────────┐
│  Streamlit UI       │  Chat view · facet score explorer ·
│  app.py             │  heatmaps · analytics
└─────────────────────┘
```

### Why this is NOT a one-shot solution

Each scoring call follows a strict **3-step chain-of-thought** process:

1. **OBSERVE** — identify specific signals in the conversation text
2. **REASON** — explain what those signals imply about the facet level
3. **CONCLUDE** — assign score (1–5) + confidence (0.0–1.0)

Facets are also split into **19 semantic clusters** (~15 facets each), so the model scores a coherent group of related facets per call — not all 276 at once.

### How it scales to 5000 facets

- **Facet registry** (`data/facets_registry.csv`): add any number of rows — zero code changes required.
- **Clusterer** dynamically computes the number of batches: `ceil(n_facets / BATCH_SIZE)`. At 5000 facets with batch size 15 → ~334 clusters per turn.
- The scoring loop iterates over however many clusters exist.

```bash
git clone <repo-url>

pip install -r requirements.txt
```

Edit `.streamlit/secrets.toml`:


## Running the pipeline

Run each step once, in order. Every step supports **resume** — safe to re-run if interrupted.

### Step 1 — Preprocess facets

Cleans the raw CSV, removes non-inferrable entries, and uses `llama-3.3-70b-versatile` to generate descriptions and score anchors for each facet.

```bash
python -m src.preprocessor
```

Output: `data/facets_registry.csv`

### Step 2 — Cluster facets

Groups the 276 inferrable facets into semantic batches of ~15 using sentence-transformer embeddings + PCA sorting.

```bash
python -m src.clusterer
```

Output: `data/facets_clusters.json`

### Step 3 — Generate conversations

Generates 50 synthetic multi-turn conversations covering diverse scenarios (customer service, emotional support, technical help, debate, safety-critical, creative, etc.).

```bash
python -m src.conversation_generator
```

Output: `conversations/conv_001.json` … `conv_050.json`, `conversations/index.json`

### Step 4 — Score conversations

Scores every turn of every conversation on all 276 facets via Groq. Uses async I/O with a rate limiter (28 req/min for Groq free tier). Saves per-conversation result files incrementally — safe to interrupt and resume.

```bash
# Score all conversations
python -m src.scorer

# Score specific conversations only
python -m src.scorer conv_001 conv_002
```

Output: `results/conv_001_scores.json` … `conv_050_scores.json`

> **Note:** Full run takes ~4 hours on the Groq free tier (7,182 API calls at 30 req/min). Monitor progress with `tail -f /tmp/scoring_run.log`.

### Step 5 — Launch the UI

```bash
streamlit run app.py
```

Opens at [http://localhost:8501](http://localhost:8501).

## Facet registry columns

| Column | Description |
|---|---|
| `facet_raw` | Original name from the CSV |
| `facet_name` | Cleaned name (no numbering, no trailing colons) |
| `inferrable` | `True` if the facet can be scored from conversation text |
| `category` | Semantic group (Personality, Emotional, Cognitive, etc.) |
| `description` | One-sentence definition for use in scoring prompts |
| `score_anchor_1` | What score 1 looks like in conversation |
| `score_anchor_3` | What score 3 looks like in conversation |
| `score_anchor_5` | What score 5 looks like in conversation |
| `polarity` | `positive` / `negative` / `neutral` |
| `weight` | Relative importance (default `1.0`, adjustable) |

## Hard constraints checklist

| Constraint | How it's met |
|---|---|
| No one-shot prompting | 3-step chain-of-thought (observe → reason → score) per cluster call |
| Open-weights ≤ 16B | Scoring uses `llama-3.1-8b-instant` (8B, Meta Llama 3 license) via Groq |
| Scales to ≥ 5000 facets | Facet registry is a plain CSV; clusterer auto-computes batch count; scorer loops over clusters — no hardcoded counts anywhere |
